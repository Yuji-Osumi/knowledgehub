# 0.5 UI/UX要件（利用シナリオ）

## 1. 基本方針
- Markdownを中心とした軽量で直感的な操作性
- 階層構造とタグで情報整理を容易に
- 最小限の画面遷移で操作可能（SPA的体験）
- レスポンシブ対応でPC・タブレット・スマホで快適に利用

## 2. 利用シナリオ例

### 2.1 記事作成・編集
1. ダッシュボードから「新規記事」をクリック
2. Markdownエディタで本文入力
3. タグを付与
4. 親フォルダを指定して階層を設定
5. 「保存」で記事作成、下書きは自動保存
6. 作成後は記事閲覧画面に遷移、Markdownプレビューを即座に確認

### 2.2 記事閲覧・検索
1. ダッシュボードから検索バーにキーワード入力
2. タグで絞り込み
3. 検索結果を階層表示
4. 記事をクリックしてMarkdown表示
5. 関連ノートがあればサイドバーに表示

### 2.3 階層管理
1. 記事一覧画面でフォルダ・記事のツリー表示
2. 新しいフォルダを作成
3. 記事をドラッグで別フォルダに移動（拡張）
4. 階層構造を折りたたみ・展開して表示

### 2.4 タグ管理
1. タグ管理画面で使用中タグを一覧表示
2. 新規タグを作成、既存タグを編集・削除
3. 記事作成時にタグを付与して整理

### 2.5 バージョン管理（拡張）
1. 記事編集画面から「履歴」をクリック
2. 過去バージョンを確認（差分表示）
3. 必要に応じて過去バージョンに戻す

## 3. 操作性・UX要件
- Markdown入力時はリアルタイムプレビュー
- 検索結果は即座に表示（1秒以内目標）
- 階層・タグの絞り込みは直感的な操作で可能
- モバイルでは折りたたみ式ツリー表示、検索バーは常設
- エラーや操作結果は明確にフィードバック表示

## 4. デザイン方針
- シンプル・ミニマルなUI
- 視認性の高いフォント・カラーリング
- Tailwind CSSやChakra UIで統一感を確保
- Markdown表示はVSCode風の色付きレンダリング
