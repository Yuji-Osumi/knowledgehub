# 5.1 画面一覧・遷移定義（UIスタブ）

目的
- バックエンド未接続で画面構成と遷移のみを検証する
- 技術前提: Vite + React + TypeScript + React Router
- ダミーデータ表示のみ（入力値検証・API通信は範囲外）

## 画面一覧（MVP最小）
| 画面     | URL                | 役割                                | 主な遷移                                                 |
| -------- | ------------------ | ----------------------------------- | -------------------------------------------------------- |
| ログイン | /login             | メール/パスワード入力、擬似ログイン | 「ログイン」→ /articles                                  |
| 記事一覧 | /articles          | 記事リスト表示（簡易・平坦）        | タイトル→ /articles/:id、新規→ /articles/new             |
| 記事詳細 | /articles/:id      | タイトル/本文/タグ表示              | 「編集」→ /articles/:id/edit、「戻る」→ /articles        |
| 記事編集 | /articles/:id/edit | 既存記事の編集（ダミー保存）        | 「保存」→ /articles/:id、「キャンセル」→ /articles       |
| 新規作成 | /articles/new      | 新規記事の作成（ダミー保存）        | 「保存」→ /articles（仮戻り）、「キャンセル」→ /articles |
| 404      | *                  | 想定外URL時の案内                   | 「一覧へ」→ /articles                                    |

## 遷移フロー
- /login → /articles（ログインボタン）
- /articles → /articles/:id（記事クリック）
- /articles → /articles/new（新規作成）
- /articles/:id → /articles/:id/edit（編集）
- /articles/:id/edit → /articles/:id（保存）／→ /articles（キャンセル）
- /articles/new → /articles（保存/キャンセルで仮戻り）

## 認証（スタブ）
- 未ログインで /articles 配下にアクセス時、/login へリダイレクト（擬似状態）
- ログイン状態はメモリ保持（Context など）で十分。永続化しない

## ルーティング規約
- 動的パラメータは :id
- 画面遷移を最優先し、フォーム値はローカル state のみで管理
- 各画面に最低1つの戻り動線を配置（一覧・詳細いずれか）

## 非対象（今回除外）
- 入力バリデーション、下書き自動保存、タグ編集、階層ツリーUI
- 検索のフィルタリング/ランキング、認可
- 実 API 連携（全てダミーデータ）

## 備考
- スタイリングは Tailwind で簡易に実装
- 画面遷移の確認が主目的のため、UI表現は最小限

### 認証リダイレクト補足
- ログイン済みの状態で `/login` にアクセスした場合は `/articles` へリダイレクトする

### エディタのプレースホルダ
- 記事本文入力は現時点では `textarea` を使用する
- リッチエディタ（Tiptap）への差し替えは後続フェーズで行う
- 本方針は「04_Frontend実装方針」に準拠する
