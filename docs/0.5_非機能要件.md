# 0.4 非機能要件定義

## 1. 性能・応答性
- 記事検索（全文検索＋タグ絞り込み）: 1000件規模のデータでレスポンス 1秒以内
- 記事一覧表示・階層ツリー表示: 500件程度の記事を表示してもスムーズ
- Markdownレンダリング: 即時プレビュー表示（遅延なし）

## 2. 可用性・信頼性
- システム稼働率 99% 以上を目標（個人開発レベル）
- データベースは定期バックアップ（週1回以上）を想定
- 記事編集時のデータ損失防止のため自動下書き保存機能

## 3. 保守性・拡張性
- Clean Architectureを採用し、フロント・バックエンドの依存関係を整理
- SQLAlchemy + AlembicによるDBマイグレーション管理
- 機能追加時も既存機能に影響が出にくいモジュール構成

## 4. セキュリティ
- パスワードはハッシュ化（bcrypt等）
- Cookieベースで認証管理
- 入力値検証とSQLインジェクション対策を実施
<!-- - 権限管理（Admin / Reader）により不正アクセス防止（不要） -->

## 5. 可観測性・運用
- ログ管理: ユーザー操作ログ、APIアクセスログを取得（簡易的）
- エラー通知: ローカル開発時はコンソール、将来的にはメール通知やSlack連携
- 開発時にDockerで環境構築可能、運用負荷を軽減

## 6. 利用環境
- ブラウザ: 最新版Chrome, Firefox, Edge, Safari
- バックエンド: Python 3.10+、FastAPI
<!-- - フロント: レスポンシブ対応（PC/タブレット/スマホ）（不要） -->

## 7. 拡張性
- 将来的にAI補助やアクセス統計などを追加可能
- PostgreSQL全文検索の拡張（GINインデックス・重み付け検索）に対応
- マルチユーザー化・チーム利用にスムーズに対応可能
