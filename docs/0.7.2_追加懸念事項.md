# 0.8 追加検討事項・懸念点リスト

## 1. 要件・機能面
- MVP範囲の粒度明確化
  - タグ機能や下書き機能の優先度をさらに具体化
- 拡張機能の実装リソース管理
  - バージョン管理、Wikiリンク、ダッシュボード、AI補助機能など段階的に追加
- 競合差別化の明文化
  - NotionやObsidianとの差別化ポイントを仕様書内に明示

## 2. 技術面
- 全文検索の負荷試算
  - 将来的なデータ増加を想定したインデックス戦略やキャッシュ検討
- AI補助機能の利用制約
  - 無料ベースの利用前提で、APIレート制限や利用上限を確認
- テスト・品質保証のカバー範囲
  - UIのE2Eテストや負荷テストも計画に入れるか検討

## 3. インフラ・デプロイ
- 運用コスト・環境再現性
  - CI/CDやステージング環境をどう作るか明文化
- セキュリティ要件の明確化
  - パスワードハッシュアルゴリズム、HTTPS必須化、JWTセキュリティ強化など

## 4. プロジェクト管理
- ドキュメント整備
  - 仕様書、ER図、画面遷移図、API仕様書を一貫管理
- スコープ管理
  - MVP → 拡張機能の段階を明確化しタスク優先度や工数見積もりに反映
