## 3. 利用上のポイント
- ドキュメントは **レビュー用の詳細版とサマリー版の2段階** を用意
- アプリは **MVP → 拡張機能** の段階的リリースを想定
- 成果物一覧は開発優先度、ワイヤーフレーム・API設計・DB設計に直結
- ポートフォリオとして提出する際は **ドキュメントとアプリの両方が揃っていることが評価ポイント**
