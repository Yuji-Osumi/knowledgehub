# 0.2 MVP・拡張機能チェックリスト

## 1. MVP（必須機能）
以下は、まず作るべき最小限の機能で、プロダクトとして価値を提供できる範囲です。

### 1.1 認証・ユーザー管理
- [ ] ユーザー登録（メール + パスワード）
- [ ] ログイン / ログアウト
- [ ] 自分の記事の閲覧・編集権限管理

### 1.2 記事管理
- [ ] 記事作成（Markdown対応）
- [ ] 記事編集（バージョン履歴はオプション）
- [ ] 記事削除
- [ ] 階層構造（フォルダ管理）
- [ ] タグ付け
- [ ] 下書き機能

### 1.3 検索
- [ ] タイトル・本文の全文検索（PostgreSQL tsvector + GIN）
- [ ] タグ絞り込み検索（AND / OR）
- [ ] 関連度順ソート（タイトル重視）
  → 検索バーは全ページ常設（スラック型）

➡ Notionより速く、Obsidianより設計自由度が高い強みを出す。

### 1.4 UI/UX
- [ ] 記事一覧（ツリー表示）
- [ ] 検索バー（画面上部常設）
- [ ] Markdownエディタ + プレビュー機能
- [ ] 基本的なレスポンシブ対応

### 1.5 軽量AI（差別化要素をMVPに）
- [ ] Embeddings を用いた“関連ノート自動推薦”（コサイン類似度でOK）

---

## 2. 拡張機能（追加で高評価・体験向上）
MVP完成後に追加すると、ポートフォリオとしての価値がさらに高まる機能です。

### 2.1 バージョン管理
- [ ] 履歴の一覧
- [ ] 記事編集差分表示（diff）
- [ ] バージョンリストからのリストア

### 2.2 ナレッジリンク
- [ ] [[記事名]] 形式の相互リンク（Notion風）
- [ ] 関連ノート提示（自動/手動）

### 2.3 タグ管理・フィルタ
- [ ] タグ一覧管理画面
- [ ] タグによる複数記事の横断検索

### 2.4 UI改善
- [ ] サイドバーでツリー構造の操作性向上
- [ ] 最近更新・人気記事のダッシュボード表示
- [ ] 検索結果のランキング改善（関連度・閲覧数）

### 2.5 ログ・監査
- [ ] 誰がいつ記事を作成/編集したかの記録
- [ ] APIアクセスログ（必要に応じて）

### 2.6 AI補助（無料ベース）
- [ ] 関連ノートの推薦（タグ・タイトル類似度ベース）
- [ ] 記事要約・簡易サマリー生成
- [ ] 記事検索のAI補正（semantic search）

### 2.7 その他
- [ ] 高速化・キャッシュ（Redisなど、必要に応じて）
- [ ] 将来的な多言語対応やテーマ切替
