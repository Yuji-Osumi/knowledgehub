# 基本方針（KnowledgeHub）

## 目的

このドキュメントは、KnowledgeHub 開発における **すべての実装判断の拠り所**となる基本方針を定める。

本プロジェクトは個人開発・MVP前提であり、
**完成させること・動かすことを最優先**とする。

## 実装における最優先事項

1. 動くこと
2. 読めること
3. 直せること

※ 将来の拡張性・汎用性は **後回し** とする

---

## 設計・実装の基本スタンス

* 「最小 → 動く → 整える」の順で進める
* 迷ったら **要件定義に立ち戻る**
* 過剰な抽象化・分割はしない
* コード量が少ない選択を優先する

---

## コード品質に対する考え方

* 「きれいさ」より「意図の明確さ」を優先
* コメントは **Why（なぜ）** を書く
* What / How はコードで表現する

---

## このドキュメントの更新ルール

* 実装を進める中で違和感が出たら更新してよい
* 更新理由は Git の commit message に残す
* 規模が大きくなった場合のみ章を分割する

---

## 最後に

この基本方針は、
**実装中に迷ったときに立ち返るための「北極星」**である。

「正しいか」よりも「前に進めるか」を判断基準とする。
