# 命名規則（KnowledgeHub）

## 目的

本ドキュメントは、KnowledgeHub における **命名の迷いを最小化する** ことを目的とする。
命名では「短さ」よりも **意味の明確さ・読み手への配慮** を優先する。

---

## 基本原則（全言語共通）

* 意味が明確に伝わる名前を使う
* 一時的な実装でも `tmp` / `hoge` は使用しない
* 省略は一般的に通じるもののみ許可する

**許可される省略例**
* `id`, `db`, `api`, `url`
* `req`, `res`（スコープが限定される場合）

---

## Boolean 命名（共通）

真偽値は **状態や可否が分かる名前** にする。

```txt
is_xxx / has_xxx / can_xxx
```

例：
* `is_published`
* `has_permission`
* `can_edit`

❌ `publish_flg`, `enable`

## Python

### ファイル・モジュール

* `snake_case.py`
* 名詞または名詞句

```txt
notes.py
auth_session.py
```

### 変数・関数

* `snake_case`
* 動詞 + 目的語

```python
def create_note():
def fetch_notes():
```

### クラス

* `PascalCase`
* 役割が分かる名前

```python
class NoteCreateRequest(BaseModel):
class NoteResponse(BaseModel):
```

### 定数

* `UPPER_SNAKE_CASE`

---

## TypeScript

### ファイル

* コンポーネント：`PascalCase.tsx`
* その他：`camelCase.ts`

```txt
NoteList.tsx
apiClient.ts
```

### 変数・関数

* `camelCase`

### コンポーネント / 型

* `PascalCase`

```tsx
export function NoteEditor() {}

type Note = {
  id: number
}
```

---

## 禁止事項（命名）

* 意味のない短縮名・数字付き変数名（`data1`, `tmp2`）
* 用途が曖昧な名前（`info`, `data`, `value`）

---

## 例外

MVP 優先で規約を破ることは許可する。
理由は以下に1行で記録する。

```txt
docs/03_実装方針/99_例外・設計判断.md
```
